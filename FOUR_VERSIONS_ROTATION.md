# 四套版本循環執行機制

## 🔄 核心概念

**當期只會有一個動畫在跑，總共有四個在循環執行**

每期開獎時，四套版本邏輯只有**一套**在執行，但四套版本輪流循環運行。

## 📊 執行流程圖

```

          │ (左側平衡版)     │ (8個數字)   │ (3顆號碼)
          └─────────────────┴──────────────┴─────────────────
          
第2024期  │ ▶ Version B 執行  │ 分析當期特徵  │ 預測 2025 期
          │ (右側單數版)     │ (8個數字)   │ (3顆號碼)
          └─────────────────┴──────────────┴─────────────────
          
第2025期  │ ▶ Version C 執行  │ 分析當期特徵  │ 預測 2026 期
          │ (中間雙數版)     │ (8個數字)   │ (3顆號碼)
          └─────────────────┴──────────────┴─────────────────
          
第2026期  │ ▶ Version D 執行  │ 分析當期特徵  │ 預測 2027 期

          └─────────────────┴──────────────┴─────────────────
          
第2027期  │ ▶ Version A 執行  │ 分析當期特徵  │ 預測 2028 期
          │ (循環回到 A)     │ (8個數字)   │ (3顆號碼)
          └─────────────────┴──────────────┴─────────────────
          
          ...循環繼續...
```

## 🎯 每期的執行步驟

### 第 N 期 (當期已開獎)

```
1️⃣ 接收當期開獎結果
   └─ 獲得 8 顆號碼 (或更多)

2️⃣ 分析當期特徵
   ├─ 區間分布: 哪些區間有號碼
   ├─ 奇偶比例: 奇數 vs 雙數
   ├─ 集中度: 號碼集中在哪裡
   ├─ 相鄰密度: 號碼是否相鄰
   └─ ... 其他 8 個特徵

3️⃣ 識別版本
   └─ 判斷當期特徵符合四套版本中的哪一套
      範例: "這期的特徵符合 Version A"

4️⃣ 根據版本預測
   └─ 用該版本的邏輯預測下期最可能的 3 顆號碼
      範例: "基於 Version A 的邏輯，預測下期是 [15, 28, 52]"
```

### 第 N+1 期 (預測期變成當期，再往前推一期)

同樣的流程再執行一遍，但識別的版本會是下一套。

## 🔍 版本識別範例

### 假設當期開獎結果: [5, 12, 25, 38, 45, 52, 61, 78]

```
分析特徵:
├─ 區間分布: [2, 2, 1, 2, 1, 0, 0, 0]  (Zone 1-8)
├─ 左側(1,3,5,7): 2+1+0+0 = 3 顆
├─ 右側(2,4,6,8): 2+2+0+0 = 4 顆
├─ 奇數個數: 4 顆 (5, 25, 45, 61)
├─ 雙數個數: 4 顆 (12, 38, 52, 78)
└─ 集中度: 0.35

識別邏輯:
├─ 左側(3) < 右側(4) ❌ → 不是 Version A
├─ 右側(4) > 左側(3) ✓ && 奇數(4) < 6 ❌ → 不是 Version B
├─ 中間區間 + 集中度 0.35 > 0.4? 接近... → 可能是 Version C
└─ 結論: Version C (中間雙數版)

預測結果:
└─ 基於 Version C 的特徵，預測下期 3 顆: [14, 48, 56]
```

## 🔐 為什麼要四套版本輪流？

1. **多樣性**: 四個不同的邏輯確保全面覆蓋
2. **輪流執行**: 不是同時判斷四套，而是序列判斷
3. **適應性**: 每套版本對應不同的開獎特徵
4. **精度**: 每套版本都針對特定特徵最佳化

## 💡 使用例子

### 在應用中

```dart
// 1. 加載最新開獎結果
final records = await bingoService.fetchRecent();

// 2. 進行預測 (自動執行上述 1-4 步驟)
final prediction = await bingoService.analyze(records);

// 3. 取得結果
print('當期號碼: ${prediction.numbers}');
print('識別版本: ${prediction.animationVersion}');  // 例: 'animation_center_even'
print('下期預測: ${prediction.animationPredicted}');  // 例: [14, 48, 56]
print('信心度: ${prediction.animationConfidence}');  // 例: 0.75 (75%)
```

## 📋 版本對照表

| 版本 | 優先區間 | 特點 | 特徵號碼 |
|------|---------|------|---------|
| **A** | 左側 (1,3,5,7) | 平衡分布 | 5, 15, 25, 35, 45, 55, 65, 75 |
| **B** | 右側 (2,4,6,8) | 單數多 | 所有奇數 (1, 3, 5, 7, ...) |
| **C** | 中間 (1,2,5,6) | 雙數多 | 所有雙數 (2, 4, 6, 8, ...) |
| **D** | 邊界 (3,4,7,8) | 分散分布 | 邊界號碼 (1, 10, 11, 20, ...) |

## ✅ 驗證方法

要驗證系統是否正確識別版本：

1. 查看 UI 上的「版本」標籤
2. 查看預測的 3 顆號碼
3. 等待下期開獎結果
4. 對比實際號碼與預測號碼
5. 觀察預測精度
