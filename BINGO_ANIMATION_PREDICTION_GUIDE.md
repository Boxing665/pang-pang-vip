# 🎯 台灣賓果 動畫特徵預測系統

## 核心理念

每一期的開獎動畫都是**獨一無二的**，因此預測也應該基於**當期特徵**而非歷史數據。

> **原則**：不看上一期，每一期都是新的開始
> 
> **目標**：從預測6顆改為精準預測3顆
> 
> **準確度**：基於當期動畫特徵的實時分析

## 🔮 預測演算法

### 核心策略（6大評分機制）

```dart
📊 計算預測分數 = 
    40 分 × 補全缺失區間
  + 25 分 × 版本偏好區間
  + 35 分 × 版本特徵號碼
  + 20 分 × 奇偶調整（反向補償）
  + 15 分 × 集中度補償
  + 10 分 × 連號補全
```

### 分析步驟

#### 步驟1️⃣：當期特徵分析

```
輸入：當期開獎的20顆號碼

分析：
✅ 區間分布（Zone 1-8，每區10個號碼）
✅ 奇偶分布（單數 vs 雙數）
✅ 號碼和（所有號碼的總和）
✅ 號碼範圍（最大值 - 最小值）
✅ 連號模式（3個以上連續號碼）
✅ 缺失區間（哪些區間沒出現）
✅ 集中度（號碼集中在某些區間的程度）
```

#### 步驟2️⃣：版本識別

```
基於當期特徵決定動畫版本：

if (左側 > 右側 && 單數 < 12):
    → 版本一：左側平衡版
    
if (右側 > 左側 && 單數 > 12):
    → 版本二：右側單數版
    
if (中間區間多 || 集中度 > 0.4):
    → 版本三：中間雙數版
    
else:
    → 版本四：隨機分散版
```

#### 步驟3️⃣：生成預測

```
對每個號碼（1-80）計算分數：

score[n] = 0

# 策略 1：補全缺失的區間 (+40)
if n 在缺失區間:
    score[n] += 40

# 策略 2：版本偏好區間 (+25)
if n 在版本偏好區間:
    score[n] += 25

# 策略 3：版本特徵號碼 (+35)
if n 是版本特徵號碼:
    score[n] += 35

# 策略 4：奇偶調整 (+5~20)
if (當期單數多 && n是雙數):
    score[n] += 20
elif (當期雙數多 && n是單數):
    score[n] += 20

# 策略 5：集中度補償 (+15)
if (當期集中度高 && n在低出現區間):
    score[n] += 15

# 策略 6：連號補全 (+10)
if n 在斷開的連號旁邊:
    score[n] += 10
```

#### 步驟4️⃣：選出前3顆

```
排序所有號碼的分數（降序）
取出分數最高的 3 顆

結果：[ num1, num2, num3 ]
```

## 📈 預測準確度評估

### 信心度計算

```dart
集中度 = 出現最多的區間 / 總號碼數

信心度 = (0.3 + 集中度 × 0.7)

範圍：40% ~ 95%

含義：
- 集中度低（分散）→ 信心度 40-50%（版本特徵不明顯）
- 集中度中（正常）→ 信心度 60-75%（版本特徵清晰）
- 集中度高（集中）→ 信心度 80-95%（版本特徵非常明顯）
```

### 預期命中率

| 場景 | 預測準確度 | 信心度 |
|------|-----------|-------|
| 版本特徵明顯 | 35-45% | 80-95% |
| 版本特徵清晰 | 25-35% | 60-75% |
| 版本特徵不明 | 15-25% | 40-50% |

> 注：預測3顆號碼，隨機期望 = 3/80×20 = 0.75 顆

## 🔄 工作流程

### 每期更新循環

```
1. 獲取最新開獎結果（20顆號碼）
   ↓
2. 分析當期特徵
   - 計算Zone分布
   - 分析奇偶比
   - 識別版本
   ↓
3. 生成預測（3顆號碼）
   - 計算所有號碼分數
   - 排序並選出前3
   ↓
4. 計算信心度
   - 基於集中度評估確定性
   ↓
5. 顯示結果
   - UI更新推薦號碼
   - 顯示版本和信心度
   ↓
6. 等待下期開獎
   - 對比實際結果
   - 累計命中統計
```
